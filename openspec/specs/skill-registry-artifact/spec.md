# skill-registry-artifact Specification

## Purpose

Define the separate, generated skill/capability registry artifact that contains the canonical skill index and Auto-invoke mappings. This artifact is treated as derived output: regenerated by `ai-specs sync`, never hand-edited.

## ADDED Requirements

### Requirement: Registry artifact is separate from AGENTS.md
The system SHALL generate a dedicated registry artifact file (e.g., `ai-specs/.skill-registry.md` or equivalent) containing the canonical skill index and Auto-invoke mappings. This file SHALL NOT be `AGENTS.md`.

#### Scenario: Sync generates registry artifact
- **WHEN** `ai-specs sync` runs against a project with skills
- **THEN** the system SHALL create or update the registry artifact file
- **AND** `AGENTS.md` SHALL NOT contain the skill index or Auto-invoke mappings

#### Scenario: Registry artifact path is configurable or conventional
- **WHEN** `ai-specs.toml` does not override the registry artifact path
- **THEN** the system SHALL use a conventional path such as `ai-specs/.skill-registry.md`

### Requirement: Registry artifact contains canonical skill index
The registry artifact SHALL list all discoverable skills with their name, description, location, and scope.

#### Scenario: Local skills are indexed
- **GIVEN** local skills exist under `ai-specs/skills/`
- **WHEN** `ai-specs sync` generates the registry artifact
- **THEN** the artifact SHALL include an entry for each local skill with name, description, and path

#### Scenario: Recipe skills are indexed
- **GIVEN** recipe-bundled skills exist under `.recipe/{recipe-id}/skills/`
- **WHEN** `ai-specs sync` generates the registry artifact
- **THEN** the artifact SHALL include an entry for each recipe skill with name, description, recipe source, and path

#### Scenario: Dependency skills are indexed
- **GIVEN** dependency skills exist under `.deps/{dep-id}/skills/`
- **WHEN** `ai-specs sync` generates the registry artifact
- **THEN** the artifact SHALL include an entry for each dependency skill with name, description, dependency source, and path

### Requirement: Registry artifact contains Auto-invoke mappings
The registry artifact SHALL include an Auto-invoke table with one row per trigger for each skill that declares `metadata.auto_invoke` and `metadata.scope`.

#### Scenario: Auto-invoke rows are emitted to registry artifact
- **GIVEN** a skill has complete `metadata.scope` and `metadata.auto_invoke`
- **WHEN** `ai-specs sync` runs
- **THEN** the registry artifact SHALL include one Auto-invoke row per trigger for matching scopes
- **AND** `AGENTS.md` SHALL NOT contain Auto-invoke rows

#### Scenario: Skills without auto-invoke metadata are omitted from Auto-invoke table
- **GIVEN** a skill lacks `metadata.auto_invoke` or `metadata.scope`
- **WHEN** `ai-specs sync` runs
- **THEN** the skill SHALL appear in the skill index
- **AND** it SHALL NOT appear in the Auto-invoke table

### Requirement: Registry artifact is derived output
The registry artifact is a derived artifact. It SHALL be regenerated by `ai-specs sync` and SHALL NOT be hand-edited. Changes to skill metadata SHALL be made in the manifest or local skill source, not in the registry artifact.

#### Scenario: Hand-edited registry artifact is overwritten on sync
- **GIVEN** the registry artifact was manually modified after sync
- **WHEN** `ai-specs sync` runs again
- **THEN** the registry artifact SHALL be regenerated from manifest and skill sources
- **AND** manual edits SHALL NOT persist

### Requirement: Registry artifact generation is idempotent
Running `ai-specs sync` multiple times with the same manifest SHALL produce byte-identical registry artifact output.

#### Scenario: Re-sync produces identical registry artifact
- **WHEN** `ai-specs sync` runs twice with the same manifest and no changes
- **THEN** the second run SHALL produce a registry artifact that is byte-identical to the first

### Requirement: Registry artifact documentation
The system SHALL document the registry artifact in project documentation, including its purpose, generation mechanism, and the rule that it must not be hand-edited.

#### Scenario: README mentions registry artifact
- **WHEN** a maintainer reads the project README or relevant docs
- **THEN** they SHALL find a description of the registry artifact
- **AND** it SHALL explain that the file is auto-generated and should not be edited manually
